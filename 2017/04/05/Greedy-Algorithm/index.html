贪心算法——习题思路

Apr 05, 2017算法学习笔记点击

内容：2017年4月5日 星期三 贪心算法。
说明：几道贪心小题的思路整理，方便对算法的理解。

Gas Station

题目

环形路线上有N个加油站，每个加油站有汽油gas[i]，从每个加油站到下一站消耗汽油cost[i]，问从哪个加油站出发能够回到起始点，如果都不能则返回-1（注意，解是唯一的）。

解题思路

每个车站可加油量和每段距离的耗油量分别存放在gas和cost中，可以设置两个变量：sum用于判断当前是否有足够的油量走到下一个加油站，若sum<0，则需将出发车站前移一个车站；remainingGas用于记录车开完整个过程后的油量。最终，判断remainingGas 是否大于零，是则返回初始车站的下标；若所有车站都出发都无法走完全程，则返回-1。

  • 贪心策略
    若remainingGas<0，即，显然无法返回终点；反之，可以。
    计算sum，设i=p为起点，若，则p到q中任意一点均不可能为起点。
    设i=q+1为起点，重复上述过程。
  • 贪心选择性：起始点一定是gas[i]-cost[i]>0 或者gas[i]-cost[i-1]>0的点，若gas[i]-cost[i]<0 或者gas[i]-cost[i-1]<0，根本无法触发。
  • 优化子结构：sum<0，则无论从当前路径p到q间哪一个点做为起点出发，都不可能走回起点。

伪代码

初始化remainingGas = 0, resultIndex = 0, sum = 0;
// 对每个车站的情况进行遍历 
for i from 0 to gas.size do
    remainingGas += gas[i] - cost[i]; 
    sum += gas[i] - cost[i]; 
    if sum < 0 do 
        sum = 0; 
        resultIndex = i + 1; 
    end if
end for
if remainingGas < 0 do return -1;    // 所有加油站的加油量小于消耗量        
else return resultIndex;            // 有解

时间复杂度为：O(n)。

最大盛水量

题目

如图，在x轴上树立了n块木板，它们的高度为H[i]。求任意两块木板和x轴组成的容器，盛水量最大是多少。

解题思路

两块木板见水的高度由较矮的那块木板决定，所以对于一块木板x来说，我们把它当成较矮的那条边时，木桶最大盛水量为：在该木板左右两侧，分别寻找所有比它高的木板中离它最远的那一块，记为l、r，maxS=max(maxS, H[x](x-l), H[x](r-x)).

如上图，若得到最大盛水量时 ，F是较短的那条边，则最大面积为[C,F]或[F,H]组成的容器。

初始化maxS=0为最大面积，我们可以遍历所有的边，依次将其当作得到最大盛水量时最短的边，然后寻找其左右两侧比它高的离它最远的边，计算得到两个面积，与manS比较。最终遍历结束可得到最大面积maxS。

  • 贪心策略
    我们可以遍历所有的边，依次将其当作得到最大盛水量时最短的边，然后寻找其左右两侧比它高的离它最远的边。
  • 贪心选择性：若在得到最大面积maxS时，x是其较短边，则最大边必为x左右两侧比他高的木板中离他最远的。（显然）

伪代码

用rank[i]分别对应H[i]中木板高度由小到大排序的序号。
暴力排序时间复杂度为O(n^2)：便利n次，第一次找最小，第二次找次小……
for i from 1 to n do
    for j from 1 to i-1 do      //找到j前从左向右第一个比它大的
        if rank[j]>rank[i], l=j,break. 
    end for
    for j from n to i+1 do      //找到j后从右向左第一个比它大的
        if rank[j]>rank[i], r=j,break. 
    end for
    maxS=max(maxS, H[i]*(i-l), H[i]*(r-i))
end for

时间复杂度T(n)=O(n^2)。

Color a Tree

题目

一棵树，结点个数为n，根结点为r。每个结点都有一个权值ci，开始时间为0，每染色一个结点需要耗时1，每个结点的染色代价为ci ∗ ti（ti 为当前的时间），每个结点只有在父结点已经被染色的条件下才能被染色。求染完整棵树需要花费的最小代价。

解题思路

在当前树中，找出权值最大的点，若这个点是根节点，则首先对他染色；若该节点为非根节点，我们贪心地认为，访问过该节点的父节点后，直接访问该节点，所需的代价最小。
故，这两个节点是紧挨着访问的，所以我们可以将这两个节点合并为一个新节点，将该节点和其父节点的所有子节点都设为新节点的子节点,该节点该节点父节点的父节点为新节点的父节点。

处理过程如下：

  1. 对于树中每个节点node[i]初始化，node[i].c=ci 为总权值（合并为这个节点的所有节点的总权值）， node[i].w=ci 为当前权值（该节点的总权值／经过该节点的时间）， node[i].t=1 为经过这个结点需要的耗时（也可以理解为这个结点包含几个合并的结点）， node[i].pre 为父结点 。
  2. 初始化染色的总代价ans=∑ci ，因为本身染色需要耗时1 ，也就要支付代价ci*1。
  3. 找到一个最大权值非根结点，将其m与其父亲p合并形成一个新的结点： node[p].c+=node[m].c ， node[p].t+=node[m].t ， node[p].w=1.0*node[p].c/node[p].t （新结点权值变成算术平均值，因为到这个结点的代价被平分分给t个结点）。
  4. 总代价ans+= node[m].c*node[p].t ，表示经过父节点p后，需要经历 node[p].t 时间才到达m ，所以讲m同p合并后，总代价要加上这段路径的代价；
  5. 重复3、4直到结点只有一个为止。
  • 贪心策略
    对于一个非根结点，如果它的权值在所有非根节点中是最大的，那我们访问过它的父节点后立即访问它，代价最小。为了能继续贪心，我们需要合并这两个节点，给每个节点赋上两个权值，node[i].t表示对第i个节点染色所需时间（第i个节点实际包含的节点数），node[i].c表示第i个节点的总权值(第i个节点实际包含的节点的权值和)。此时，定义一个节点的当前权值等于node[i].w为node[i].c/node[i].t。可以证明在新的权值定义下，以上贪心同样成立。
    因此我们每次查找权值最大的那个节点，然后向根开始归并，直接最终只剩下一个根位置。
  • 证明贪心选择性和优化子结构。
    时间是每次加1的，不妨设c1 < c2，t1 < t2，则代价最小的方案为c1t2+c2t1。（显然）
    则在整个时间序列中，代价最大的是每次选择代价最小的节点，代价最小的是每次选择代价最大的节点。

伪代码

初始化node node[i].c=ci, node[i].w=ci, node[i].t=1, node[i].pre=该节点的父节点, ans=∑ci
for cnt from 1 to n do
    遍历当前树，找出当前权值node[m].w最大的非根节点m   //时间复杂度O(n)
    node[m].w = 0; //把找到的最大权值节点的权值变为0，以免影响后面求最大节点的搜索
    p = node[m].pre;
    ans += node[m].c * node[p].t
    遍历当前树，将所有指向m节点的子节点的父节点该层p节点 //时间复杂度O(n)
    node[p].t += node[m].t;                 //  对总步数和总时间相加
    node[p].c += node[m].c;                 //  节点的权值
    node[p].w = 1.0*node[p].c/node[p].t;    //  求平均的权值
end for

时间复杂度T(n)=O(n^2)。

石子问题

题目

有n个石子，每个石子都有它自己的重量wi ，我们要将它们聚集成一堆，要求每次只能操作两堆，每次操作的代价是操作石子重量的和，那么请设计一个方案使聚集的总代价最小。

解题思路

  • 贪心策略
    由于每次只能操作两堆，每次操作的代价是石子的重量之和，我们当然希望越重的操作次数越少，所以每次选择当前所有堆中最轻的两个堆合并。
  • 贪心选择性：设x、y是最轻的两个堆，另有一堆z，它们的重量分别是wz、wy、wz，若先合并x、y，则代价为cost1=(wx+wy)+(wx+wy+wz) ，若先合并y、z，cost2=(wy+wz)+(wx+wy+wz)>cost1。
  • 优化子结构：设x、y合并为一个新堆z，T是合并前的最优方案，则T’=T-{x,y}一定是C’=C-{x,y}+{z}的最优方案，其中C为原石子堆。
    原代价为WT，新代价为WT’，设第i个堆移动了di次，有dz=dx-1=dy-1。
    WT-WT’=w(x)d(x)+w(y)d(y)-w(x+y)d(x+y)= w(x)d(x)+w(y)d(y)-w(x+y)d(z)= w(x)d(x)+w(y)d(y)-(w(x)+w(y))(d(x)-1)=w(x)+w(y)
    WT为原最优方案，w(x)、w(y)为最小重量石子。
    所以WT’为最优方案。

伪代码

过程类似于哈夫曼树的构建


sum = 0
for cnt from 0 to n-1 do
    从数组中找出最小的两个w[i],w[j]    //O(n)
    sum += w[i]+w[j]
    将w[i] = w[i]+w[j],w[j]设为一个非常大的数//w[i]相当于合并后的新节点
end for

时间复杂度O(n^2)。
